# 论文分享系统设计文档

**日期：** 2026-03-08
**状态：** 已批准

## 概述

设计一个论文分享系统，允许用户将生成的论文总结公开到画廊，供所有人浏览。用户可以选择是否公开，公开的论文会有显著的标识。

## 核心需求

1. 用户可以选择将论文公开到画廊
2. 只有成功生成白板图的论文才能被分享
3. 公开的论文在详情页有显著标识
4. 提供公开画廊页面，展示所有公开的论文
5. 画廊采用图片卡片布局，以白板图为主视觉

## 技术方案

### 方案选择：简单字段方案

在 `papers` 表添加字段，无需额外的分享表。优点是实现简单、查询性能好、易于维护。

## 数据库设计

### 修改 `papers` 表

添加两个新字段：

```typescript
export const papers = sqliteTable(
  "papers",
  {
    // ... 现有字段
    isPublic: integer("is_public", { mode: "boolean" })
      .notNull()
      .default(false),
    publishedAt: integer("published_at", { mode: "timestamp" }),
  },
  (table) => ({
    // ... 现有索引
    publicIdx: index("papers_public_idx").on(
      table.isPublic,
      table.publishedAt,
    ),
  }),
);
```

**字段说明：**
- `isPublic`: 是否公开到画廊，默认 `false`
- `publishedAt`: 公开时间，用于画廊排序，默认 `null`

**索引优化：**
- 添加复合索引 `papers_public_idx` 优化画廊查询性能

### 数据库迁移

需要创建迁移文件添加这两个字段和索引。

## API 设计

### 新增 tRPC 路由

#### 1. `paper.togglePublic`

切换论文的公开状态。

**输入：**
```typescript
{
  paperId: string
}
```

**权限：**
- 仅论文创建者可调用
- 论文必须满足分享条件（status === 'completed' 且有白板图）

**逻辑：**
```typescript
if (isPublic) {
  // 取消公开
  isPublic = false
  publishedAt = null
} else {
  // 公开
  isPublic = true
  publishedAt = new Date()
}
```

**返回：**
```typescript
{
  success: boolean
  isPublic: boolean
}
```

#### 2. `paper.listPublic`

获取公开画廊的论文列表。

**输入：**
```typescript
{
  page: number      // 页码，从 1 开始
  limit: number     // 每页数量，默认 20
}
```

**权限：**
- 所有人可访问（包括未登录用户）

**查询条件：**
```typescript
WHERE isPublic = true
  AND status = 'completed'
  AND deletedAt IS NULL
ORDER BY publishedAt DESC
```

**返回：**
```typescript
{
  papers: Array<{
    id: string
    title: string
    whiteboardImageR2Key: string
    publishedAt: Date
    // 不返回 userId 等敏感信息
  }>
  total: number
  page: number
  limit: number
}
```

#### 3. `paper.getById` (修改现有)

修改权限检查逻辑。

**权限检查：**
```typescript
// 允许访问的条件：
// 1. 是论文创建者
// 2. 或者论文是公开的
if (paper.userId !== session.user.id && !paper.isPublic) {
  throw new Error('Unauthorized')
}
```

### 1. 论文详情页 (`/papers/$paperId`)

#### 1.1 公开状态横幅（仅创建者可见）

**位置：** 页面顶部，面包屑导航下方

**未公开状态：**
```

│ 🌍 分享到公开画廊                            │
│                                             │
│ 让更多人看到你的论文总结                     │
│                                             │
│ [分享到画廊] 按钮                            │
└─────────────────────────────────────────────┘
```

**已公开状态：**
```
┌─────────────────────────────────────────────┐
│ ✓ 此论文已公开分享                           │
│                                             │
│ 分享链接: https://picx.app/papers/xxx       │
│ [复制链接] [取消公开]                        │
└─────────────────────────────────────────────┘
```

**按钮状态：**
- 如果 `status !== 'completed'` 或 `!whiteboardImageR2Key`：
  - 按钮禁用

#### 1.2 公开标识（所有访客可见）

**位置：** 侧边栏论文信息卡片中，状态徽章下方

**样式：**
```
┌─────────────────────┐
│ 状态: [已完成]       │
│ 🌍 公开分享          │
└─────────────────────┘
```

使用地球图标 + "公开分享"文字，样式与现有徽章一致。

#### 3.2 响应式布局

- **移动端：** 1 列
- **平板：** 2 列
- **桌面：** 3-4 列

#### 3.3 卡片设计

**卡片内容：**
- 白板图片（主视觉，占据卡片 70-80% 空间）
- 论文标题（图片下方或覆盖在图片底部）
- 分享时间（相对时间，如"2天前"）

**交互效果：**
- 悬停：卡片轻微上浮 + 阴影加深
- 点击：跳转到 `/papers/$paperId`

**视觉风格：**
- 白板图片：圆角 `rounded-xl`，带阴影
- 卡片背景：`var(--parchment)`
- 标题：`font-serif`，学术风格
- 间距：卡片间距适中，营造画廊感

## 错误处理

### 权限错误
- 非创建者尝试切换公开状态：返回 403 错误
- 访问未公开且非自己的论文：返回 403 错误

### 业务逻辑错误
- 尝试分享未完成的论文：返回错误提示
- 尝试分享没有白板图的论文：返回错误提示

## 性能考虑

### 数据库查询优化
- 使用复合索引 `papers_public_idx` 优化画廊查询
- 画廊查询只返回必要字段，不返回敏感信息

### 图片加载优化
- 白板图片使用 R2 的 CDN 加速
- 考虑添加图片懒加载（Intersection Observer）
- 可选：生成缩略图版本用于画廊展示

## 安全考虑

### 隐私保护
- 公开画廊不显示论文创建者信息
- 不返回 userId 等敏感字段

### 访问控制
- 严格检查论文所有权
- 公开论文允许所有人访问（包括未登录用户）

## 未来扩展

当前设计为未来功能预留了空间：

1. **统计功能**：可以添加独立的统计表记录浏览量、点赞数
2. **分类标签**：可以添加标签系统，支持按领域分类
3. **搜索功能**：可以添加全文搜索，支持按标题、摘要搜索
4. **热门排序**：基于浏览量、点赞数等指标排序

这些功能可以在不修改核心数据结构的情况下逐步添加。
